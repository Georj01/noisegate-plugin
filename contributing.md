# Contributing

Pull requests are welcome. For major changes, please open an issue first to discuss what you would like to change. 

## Guidelines

1. **Minimalism:** Keep the codebase flat and avoid over-engineering. Do not add unnecessary folders or dependencies.
2. **Explicit Naming:** Use clear and explicit variable names. Avoid single-letter variables like `x`.
3. **English Only:** All code, documentation, and commits must be in English.
4. **Defensive Programming:** Validate data before mapping or rendering.
5. **Kebab-Case:** Use kebab-case for all files and directories.
